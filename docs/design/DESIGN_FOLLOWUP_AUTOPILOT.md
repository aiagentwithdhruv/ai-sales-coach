# Design Document: Follow-up Autopilot

**Feature:** Follow-up Autopilot  
**Priority:** P0 (Phase 1)  
**Designer:** AI Sales Coach Team  
**Last Updated:** 2026-02-01  

---

## 1. Overview

### 1.1 Purpose
Enable sales reps to automatically generate and send contextual follow-up emails after sales calls, reducing manual work and improving conversion rates.

### 1.2 User Goals
- Save 5+ minutes per follow-up email
- Never forget to follow up
- Maintain personalized messaging at scale
- Get manager approval when needed

### 1.3 Design Principles
- **Effortless**: One-click generation from call context
- **Trustworthy**: Always show what will be sent
- **Flexible**: Edit before sending
- **Safe**: Approval workflow for sensitive sends

## 5. Interaction Patterns

### 5.1 Generate Follow-up

1. User clicks "Generate Follow-up" from Call Analyzer or manually
2. Loading state: Shimmer skeleton in editor
3. AI generates content (2-3 seconds)
4. Content populates with fade-in animation
5. Key points chips appear below editor
6. User can edit, regenerate, or accept

### 5.2 Edit Flow

1. Click on any text to edit inline
2. Changes auto-save every 5 seconds
3. "Regenerate" clears and regenerates from scratch
4. "Undo" reverts last change (CMD+Z)

### 5.3 Send Flow

**Send Now:**
1. Click "Send Now"
2. Confirmation toast: "Sending..."
3. Success toast: "Email sent to sarah@acme.com"
4. Card moves to "Sent" tab

**Schedule:**
1. Click "Schedule" dropdown
2. Select time or pick custom
3. Confirmation: "Scheduled for Tomorrow 9:00 AM"
4. Card shows scheduled badge

## 7. Responsive Behavior

### Desktop (1024px+)
- Full sidebar visible
- Editor modal: 640px width
- Stats row: 4 columns
- Follow-up list: Full width cards

### Tablet (768px - 1023px)
- Collapsible sidebar
- Editor modal: 90% width
- Stats row: 2x2 grid
- Follow-up list: Full width cards

### Mobile (< 768px)
- Bottom navigation
- Editor: Full screen sheet
- Stats row: Horizontal scroll
- Follow-up cards: Stacked, swipe actions

---

## 8. Accessibility

- All buttons have aria-labels
- Focus traps in modals
- Escape closes modal
- Tab navigation through editor fields
- Screen reader announces send confirmation
- Color is not only indicator of status (icons + text)
